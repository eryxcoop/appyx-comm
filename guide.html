Requesters¶

RemoteRequester is class that represents a requester. It is used to make requests to the api. You can create your own requester by extending this class.

1// We create remote requester
2const authorizationManager = new AppAuthorizationManager(this);
3const remoteRequester = new RemoteRequester(remoteApiUrl, authorizationManager);

FakeRequester can be used if you are testing your application without having a server running on the other side. Fake will respond the defaultResponse you indicate in your expected response.

1// We create fake requester
2const remoteRequester = new FakeRequester();

Endpoints¶

When you create and Endpoint you are defining where the request should be made, what type of request is it and what kind of responses should you expect. Your ApiClient will be able to execute the endpoint you created. For example, you can create an endpoint like this:

 1  class ExampleEndpoint extends Endpoint{
 2    url() {
 3      return "example_url";
 4    }
 5
 6    ownResponses() {
 7      return [GetExampleResponse];
 8    }
 9
10    method() {
11      return this.constructor.getMethod();
12    }
13
14    needsAuthorization() {
15      return true;
16    }
17  }
18
19
20  // Now you can use it like this
21  const endpointToExample = new ExampleEndpoint();
22  const response = await client._callEndpoint(endpointToExample);

Responses¶

Every time you make a call to the api, you will (usually) receive a response. Response is here for you to represent that response. That way you can easily tell your endpoint what kind of responses it should expect. You can use already created responses in the library such as SuccessfulApiResponse or you can create your own to make use of its full potential and ask your response for specific information

Here is an example of a response:}

 1  // We extend from successful api response because we are representing the successful response of our call.
 2  class ExampleSuccessfulResponse extends SuccessfulApiResponse{
 3    // We leave an example of the type of response you expect
 4    static defaultResponse() {
 5      return {
 6        object: {
 7          example: {
 8            value: 1,
 9          },
10        },
11        errors: [],
12      };
13    }
14
15    get exampleValue() {
16      return this.content().example.value;
17    }
18  }

Response Handler¶

Basic use¶

Response handlers are used to handle responses received from api requests. Every time you make a call you can indicate the way every expected response should be handled. For example, you can indicate that every successful response should be handled by running a function. Note that ResponseHandler is an immutable object, so every time you indicate the way responses should be handled, you are actually overriding the default response handler.

1  const customResponseHandler = ApiResponseHandler.for(
2      SuccessfulApiResponse,
3      (response, request) => {
4          doSomething();
5      },
6  );
7
8  // doSomething() will be called if exampleEndpoint returns a SuccessfulApiResponse
9  await client.exampleEndpoint(customResponseHandler);

Sometimes you want to handle an specific response always in the same way. For example, you may want to handle every 401 response by authenticating the user again. To do this you have to indicate all your general responses handlers in the client constructor. For example:

1  const generalResponsesHandler = ApiResponseHandler.for(
2      AuthenticationErrorResponse,
3      (response, request) => {
4          return authenticateUserAgain();
5      },
6  );
7  const client = new ExampleApiClient(requester, generalResponsesHandler);

Multiple response handler¶

In order to have multiple responses to consider, you can clarify them all with the handler method. Remember that ApiResponseHandler is an unmutable object, so everytime a response is add it will return a new ApiResponseHandler object. For example:

 1  let responsesHandler = new ApiResponseHandler();
 2  responsesHandler = responseHandler.handles(
 3      SuccessfulApiResponse,
 4      (response, request) => {
 5          return doSomething();
 6      },
 7  );
 8  responsesHandler = responseHandler.handles(
 9      AuthenticationErrorResponse,
10      (response, request) => {
11          return authenticateUserAgain();
12      },
13  );
14  const client = new ExampleApiClient(requester, responsesHandler);

Now you may be wondering, what happens if I want to handle a specific response in a different way? Well, you can already do that! every time you indicate the way responses should be handled, you are actually overriding the default response handler. So, if you want to handle a specific response in a different way, just override the default response handler again.

Successful and error responses¶

Sometimes you may want to have only two different cases to handle, successful responses and error responses. You can do that by:

 1  let responsesHandler = new ApiResponseHandler();
 2  responsesHandler = responseHandler
 3                      .handlesSuccess((response, request) => {
 4                          return doSomething();
 5                      },
 6                      ).handlesError((response, request) => {
 7                              return authenticateUserAgain();
 8                          },
 9                      );
10  const client = new ExampleApiClient(requester, responsesHandler);

ApiClient¶

Finally, you can create your own ApiClient by extending the ApiClient class. This class is the one that will be used to make the requests to the api.

 1  class MyApiClient extends ApiClient{
 2
 3    registerNewUser(email, password, name) {
 4      const endpoint = new RegisterUserEndpoint();
 5      return this._callEndpoint(endpoint, { email, password, name });
 6    }
 7  }
 8
 9  // We create remote requester
10  const authorizationManager = new AppAuthorizationManager(this);
11  const remoteRequester = new RemoteRequester(remoteApiUrl, authorizationManager);
12
13  // We create responses handler
14  const apiResponseHandler = new ApiResponseHandler(this);
15
16  // We create the client
17  const endpointToExample = new MyApiClient(remoteRequester, responsesHandler);
18
19  // We use it to register a new user
20  const response = await client.registerNewUser(endpointToExample);
